## Thread 1 — Weather-market structural edge (ensemble spread + NO-bias brackets)

(memory: "0W/7L YES" pattern). Three Apr 14–19 item clusters point at

**Mechanism hypothesis:** Weather-bracket markets price each bucket
independently, so collective YES+NO can deviate from $1.00 when ensemble
forecasts disagree. The NO-side is asymmetrically cheap because retail
buys "temperature will hit X" (YES), leaving structural premium on the
other side. WeatherBot already captures this implicitly; ECMWF vs NOAA
disagreement is a pre-trade edge amplifier.

**Key SCT-AUTO items:** 675 (YES+NO <$1.00 brackets, 8.1), 753 (ECMWF
spread filter, 7.6), 752 (NOAA vs ECMWF backtest, 7.6), 751 (ECMWF API
integration, 7.6), 705 (NOAA latency benchmark, 8.2), 592/593/594
(openclaw weighting / NO-side filter / alerts, 7.9 each).

**Plausible upside:** ~$3–8/day uplift on WeatherBot paper *if the
ECMWF disagreement filter lifts current paper P&L above breakeven*.
Calibrated against openclaw's published 74% WR (external reference,
not own backtest). Caveat: openclaw number is their self-report; own
Brier/WR must be measured before trusting.

**Cost to test:** 8–12 operator hours (ECMWF Copernicus API setup + NOAA
dual-pipe + 14-day paper comparison). Capital: $0 (paper). Days to
validate: 14 (WeatherBot trade cadence is slow).

**Opportunity cost:** Displaces ~10 hrs that would otherwise go to
WeatherBot live-deployment prep OR to CopyBot spread-gate maintenance.
WeatherBot is already close to a gate evaluation, so this is the
natural next increment rather than a new project.

**Falsifiability check:** Run 752 first — if NOAA-only vs ECMWF
ensemble shows no Brier-score improvement on historical resolved
